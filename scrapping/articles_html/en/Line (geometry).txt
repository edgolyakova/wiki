<p>In <a href="/wiki/Geometry" title="Geometry">geometry</a>, a <b>line</b> is an infinitely long object with no width, depth, or <a href="/wiki/Curvature" title="Curvature">curvature</a>. Thus, lines are <a href="/wiki/One-dimensional_space" title="One-dimensional space">one-dimensional objects</a>, though they may exist in <a href="/wiki/Two-dimensional_Euclidean_space" title="Two-dimensional Euclidean space">two</a>, <a href="/wiki/Three-dimensional_space" title="Three-dimensional space">three</a>, or higher <a href="/wiki/Dimension" title="Dimension">dimension</a> spaces. The word <i>line</i> may also refer to a <a href="/wiki/Line_segment" title="Line segment">line segment</a> in everyday life, which have two <a href="/wiki/Point_(geometry)" title="Point (geometry)">points</a> to denote its ends. Lines can be referred by two points that lay on it (e.g., <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\textstyle {\overleftrightarrow {AB}}}" xmlns="http://www.w3.org/1998/Math/MathML">

</p><p><a href="/wiki/Euclid" title="Euclid">Euclid</a> described a line as "breadthless length" which "lies equally with respect to the points on itself"; he introduced several <a class="mw-redirect" href="/wiki/Postulate" title="Postulate">postulates</a> as basic unprovable properties from which he constructed all of geometry, which is now called <a href="/wiki/Euclidean_geometry" title="Euclidean geometry">Euclidean geometry</a> to avoid confusion with other geometries which have been introduced since the end of the 19th century (such as <a href="/wiki/Non-Euclidean_geometry" title="Non-Euclidean geometry">non-Euclidean</a>, <a href="/wiki/Projective_geometry" title="Projective geometry">projective</a> and <a href="/wiki/Affine_geometry" title="Affine geometry">affine geometry</a>).
</p><p>In modern mathematics, given the multitude of geometries, the concept of a line is closely tied to the way the geometry is described. For instance, in <a href="/wiki/Analytic_geometry" title="Analytic geometry">analytic geometry</a>, a line in the plane is often defined as the set of points whose coordinates satisfy a given <a href="/wiki/Linear_equation" title="Linear equation">linear equation</a>, but in a more abstract setting, such as <a href="/wiki/Incidence_geometry" title="Incidence geometry">incidence geometry</a>, a line may be an independent object, distinct from the set of points which lie on it.
</p><p>When a geometry is described by a set of <a href="/wiki/Axiom" title="Axiom">axioms</a>, the notion of a line is usually left undefined (a so-called <a href="/wiki/Primitive_notion" title="Primitive notion">primitive</a> object). The properties of lines are then determined by the axioms which refer to them. One advantage to this approach is the flexibility it gives to users of the geometry. Thus in <a href="/wiki/Differential_geometry" title="Differential geometry">differential geometry</a>, a line may be interpreted as a <a href="/wiki/Geodesic" title="Geodesic">geodesic</a> (shortest path between points), while in some <a href="/wiki/Projective_geometry" title="Projective geometry">projective geometries</a>, a line is a 2-dimensional vector space (all linear combinations of two independent vectors). This flexibility also extends beyond mathematics and, for example, permits physicists to think of the path of a light ray as being a line.
</p>

<h2><span class="mw-headline" id="Properties">Properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=1" title="Edit section: Properties">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>When geometry was first formalised by <a href="/wiki/Euclid" title="Euclid">Euclid</a> in the <i><a href="/wiki/Euclid%27s_Elements" title="Euclid's Elements">Elements</a></i>, he defined a general line (straight or curved) to be "breadthless length" with a straight line being a line "which lies evenly with the points on itself".<sup class="reference" id="cite_ref-:0_1-0"><a href="#cite_note-:0-1">[1]</a></sup><sup class="reference nowrap"><span title="Page: 291">: 291 </span></sup> These definitions serve little purpose, since they use terms which are not by themselves defined. In fact, Euclid himself did not use these definitions in this work, and probably included them just to make it clear to the reader what was being discussed. In modern geometry, a line is simply taken as an undefined object with properties given by <a href="/wiki/Axiom" title="Axiom">axioms</a>,<sup class="reference" id="cite_ref-:0_1-1"><a href="#cite_note-:0-1">[1]</a></sup><sup class="reference nowrap"><span title="Page: 95">: 95 </span></sup> but is sometimes defined as a set of points obeying a linear relationship when some other fundamental concept is left undefined.
</p><p>In an <a href="/wiki/Axiom" title="Axiom">axiomatic</a> formulation of Euclidean geometry, such as that of <a href="/wiki/Hilbert%27s_axioms" title="Hilbert's axioms">Hilbert</a> (Euclid's original axioms contained various flaws which have been corrected by modern mathematicians),<sup class="reference" id="cite_ref-:0_1-2"><a href="#cite_note-:0-1">[1]</a></sup><sup class="reference nowrap"><span title="Page: 108">: 108 </span></sup> a line is stated to have certain properties which relate it to other lines and <a href="/wiki/Point_(geometry)" title="Point (geometry)">points</a>. For example, for any two distinct points, there is a unique line containing them, and any two distinct lines intersect in at most one point.<sup class="reference" id="cite_ref-:0_1-3"><a href="#cite_note-:0-1">[1]</a></sup><sup class="reference nowrap"><span title="Page: 300">: 300 </span></sup> In two <a href="/wiki/Dimension" title="Dimension">dimensions</a> (i.e., the Euclidean <a class="mw-redirect" href="/wiki/Plane_(mathematics)" title="Plane (mathematics)">plane</a>), two lines which do not intersect are called <a href="/wiki/Parallel_(geometry)" title="Parallel (geometry)">parallel</a>. In higher dimensions, two lines that do not intersect are parallel if they are contained in a <a href="/wiki/Plane_(geometry)" title="Plane (geometry)">plane</a>, or <a href="/wiki/Skew_lines" title="Skew lines">skew</a> if they are not.
</p><p>On an <a class="mw-redirect" href="/wiki/Euclidean_plane" title="Euclidean plane">Euclidean plane</a>, a line can be represented as a boundary between two regions.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference nowrap"><span title="Page: 104">: 104 </span></sup> Any collection of finitely many lines partitions the plane into <a href="/wiki/Convex_polygon" title="Convex polygon">convex polygons</a> (possibly unbounded); this partition is known as an <a href="/wiki/Arrangement_of_lines" title="Arrangement of lines">arrangement of lines</a>.
</p>
<h3><span class="mw-headline" id="In_higher_dimensions">In higher dimensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=2" title="Edit section: In higher dimensions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

</p>
<h4><span class="mw-headline" id="Collinear_points">Collinear points</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=3" title="Edit section: Collinear points">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Collinearity" title="Collinearity">Collinearity</a></div>

<p>However, there are other notions of distance (such as the <a class="mw-redirect" href="/wiki/Manhattan_distance" title="Manhattan distance">Manhattan distance</a>) for which this property is not true.
</p><p>In the geometries where the concept of a line is a <a href="/wiki/Primitive_notion" title="Primitive notion">primitive notion</a>, as may be the case in some <a href="/wiki/Synthetic_geometry" title="Synthetic geometry">synthetic geometries</a>, other methods of determining collinearity are needed.
</p>
<h3><span class="mw-headline" id="Types">Types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=4" title="Edit section: Types">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright"><div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Tangent_to_a_curve.svg"><img alt="see caption" class="thumbimage" data-file-height="280" data-file-width="400" decoding="async" height="154" src="//upload.wikimedia.org/wikipedia/commons/thumb/0/0f/Tangent_to_a_curve.svg/220px-Tangent_to_a_curve.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/0/0f/Tangent_to_a_curve.svg/330px-Tangent_to_a_curve.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/0/0f/Tangent_to_a_curve.svg/440px-Tangent_to_a_curve.svg.png 2x" width="220"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Tangent_to_a_curve.svg" title="Enlarge"></a></div>Tangent to a curve. The red line is tangential to the curve at the point marked by a red dot.</div></div></div>
<p>In a sense,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> all lines in Euclidean geometry are equal, in that, without coordinates, one can not tell them apart from one another. However, lines may play special roles with respect to other objects in the geometry and be divided into types according to that relationship. For instance, with respect to a <a href="/wiki/Conic_section" title="Conic section">conic</a> (a <a href="/wiki/Circle" title="Circle">circle</a>, <a href="/wiki/Ellipse" title="Ellipse">ellipse</a>, <a href="/wiki/Parabola" title="Parabola">parabola</a>, or <a href="/wiki/Hyperbola" title="Hyperbola">hyperbola</a>), lines can be:
</p>
<ul><li><a class="mw-redirect" href="/wiki/Tangent_line" title="Tangent line">tangent lines</a>, which touch the conic at a single point;</li>
<li><a href="/wiki/Secant_line" title="Secant line">secant lines</a>, which intersect the conic at two points and pass through its interior;<sup class="reference" id="cite_ref-cag_6-0"><a href="#cite_note-cag-6">[6]</a></sup></li>
<li>exterior lines, which do not meet the conic at any point of the Euclidean plane; or</li>
<li>a <a class="mw-redirect" href="/wiki/Directrix_of_a_conic_section" title="Directrix of a conic section">directrix</a>, whose distance from a point helps to establish whether the point is on the conic.</li></ul>
<p>In the context of determining <a href="/wiki/Parallel_(geometry)" title="Parallel (geometry)">parallelism</a> in Euclidean geometry, a <a href="/wiki/Transversal_(geometry)" title="Transversal (geometry)">transversal</a> is a line that intersects two other lines that may or not be parallel to each other.
</p><p>For more general <a href="/wiki/Algebraic_curve" title="Algebraic curve">algebraic curves</a>, lines could also be:
</p>
<ul><li><i>i</i>-secant lines, meeting the curve in <i>i</i> points counted without multiplicity, or</li>
<li><a href="/wiki/Asymptote" title="Asymptote">asymptotes</a>, which a curve approaches arbitrarily closely without touching it.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></li></ul>
<p>With respect to <a class="mw-redirect" href="/wiki/Euclidean_triangle" title="Euclidean triangle">triangles</a> we have:
</p>
<ul><li>the <a href="/wiki/Euler_line" title="Euler line">Euler line</a>,</li>
<li>the <a href="/wiki/Simson_line" title="Simson line">Simson lines</a>, and</li>
<li><a href="/wiki/Central_line_(geometry)" title="Central line (geometry)">central lines</a>.</li></ul>
<p>For a <a href="/wiki/Convex_polygon" title="Convex polygon">convex</a> <a href="/wiki/Quadrilateral" title="Quadrilateral">quadrilateral</a> with at most two parallel sides, the <a href="/wiki/Newton_line" title="Newton line">Newton line</a> is the line that connects the midpoints of the two <a href="/wiki/Diagonal" title="Diagonal">diagonals</a>.<sup class="reference" id="cite_ref-Alsina_8-0"><a href="#cite_note-Alsina-8">[8]</a></sup>
</p><p>For a <a href="/wiki/Hexagon" title="Hexagon">hexagon</a> with vertices lying on a conic we have the <a class="mw-redirect" href="/wiki/Pascal_line" title="Pascal line">Pascal line</a> and, in the special case where the conic is a pair of lines, we have the <a href="/wiki/Pappus%27s_hexagon_theorem" title="Pappus's hexagon theorem">Pappus line</a>.
</p><p><a href="/wiki/Parallel_(geometry)" title="Parallel (geometry)">Parallel lines</a> are lines in the same plane that never cross. <a class="mw-redirect" href="/wiki/Line-line_intersection" title="Line-line intersection">Intersecting lines</a> share a single point in common. Coincidental lines coincide with each other—every point that is on either one of them is also on the other.
</p><p><a class="mw-redirect" href="/wiki/Perpendicular_lines" title="Perpendicular lines">Perpendicular lines</a> are lines that intersect at <a href="/wiki/Right_angle" title="Right angle">right angles</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup>
</p><p>In <a href="/wiki/Three-dimensional_space" title="Three-dimensional space">three-dimensional space</a>, <a href="/wiki/Skew_lines" title="Skew lines">skew lines</a> are lines that are not in the same plane and thus do not intersect each other.
</p>
<h3><span class="mw-headline" id="In_axiomatic_systems">In axiomatic systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=5" title="Edit section: In axiomatic systems">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The concept of line is often considered in geometry as a <a href="/wiki/Primitive_notion" title="Primitive notion">primitive notion</a> in <a href="/wiki/Axiomatic_system" title="Axiomatic system">axiomatic systems</a>,<sup class="reference" id="cite_ref-:0_1-4"><a href="#cite_note-:0-1">[1]</a></sup><sup class="reference nowrap"><span title="Page: 95">: 95 </span></sup> meaning it is not being defined by other concepts.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> In those situations where a line is a defined concept, as in <a class="mw-redirect" href="/wiki/Coordinate_geometry" title="Coordinate geometry">coordinate geometry</a>, some other fundamental ideas are taken as primitives. When the line concept is a primitive, the behaviour and properties of lines are dictated by the <a href="/wiki/Axiom" title="Axiom">axioms</a> which they must satisfy.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2022)">citation needed</span></a></i>]</sup>
</p><p>In a non-axiomatic or simplified axiomatic treatment of geometry, the concept of a primitive notion may be too abstract to be dealt with. In this circumstance, it is possible to provide a <i>description</i> or <i>mental image</i> of a primitive notion, to give a foundation to build the notion on which would formally be based on the (unstated) axioms. Descriptions of this type may be referred to, by some authors, as definitions in this informal style of presentation. These are not true definitions, and could not be used in formal proofs of statements. The "definition" of line in <a href="/wiki/Euclid%27s_Elements" title="Euclid's Elements">Euclid's Elements</a> falls into this category.<sup class="reference" id="cite_ref-:0_1-5"><a href="#cite_note-:0-1">[1]</a></sup><sup class="reference nowrap"><span title="Page: 95">: 95 </span></sup> Even in the case where a specific geometry is being considered (for example, <a href="/wiki/Euclidean_geometry" title="Euclidean geometry">Euclidean geometry</a>), there is no generally accepted agreement among authors as to what an informal description of a line should be when the subject is not being treated formally.
</p>

<h3><span class="mw-headline" id="Projective_geometry">Projective geometry</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=13" title="Edit section: Projective geometry">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Further information: <a href="/wiki/Geodesic" title="Geodesic">Geodesic</a></div>
<div class="thumb tright"><div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Great_circle_hemispheres.png"><img alt="" class="thumbimage" data-file-height="582" data-file-width="634" decoding="async" height="202" src="//upload.wikimedia.org/wikipedia/commons/thumb/0/04/Great_circle_hemispheres.png/220px-Great_circle_hemispheres.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/0/04/Great_circle_hemispheres.png/330px-Great_circle_hemispheres.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/0/04/Great_circle_hemispheres.png/440px-Great_circle_hemispheres.png 2x" width="220"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Great_circle_hemispheres.png" title="Enlarge"></a></div>A great circle divides the sphere in two equal hemispheres, while also satisfying the "no curvature" property.</div></div></div>
<p>In many models of <a href="/wiki/Projective_geometry" title="Projective geometry">projective geometry</a>, the representation of a line rarely conforms to the notion of the "straight curve" as it is visualised in Euclidean geometry. In <a href="/wiki/Elliptic_geometry" title="Elliptic geometry">elliptic geometry</a> we see a typical example of this.<sup class="reference" id="cite_ref-:0_1-6"><a href="#cite_note-:0-1">[1]</a></sup><sup class="reference nowrap"><span title="Page: 108">: 108 </span></sup> In the spherical representation of elliptic geometry, lines are represented by <a href="/wiki/Great_circle" title="Great circle">great circles</a> of a sphere with diametrically opposite points identified. In a different model of elliptic geometry, lines are represented by Euclidean <a href="/wiki/Plane_(geometry)" title="Plane (geometry)">planes</a> passing through the origin. Even though these representations are visually distinct, they satisfy all the properties (such as, two points determining a unique line) that make them suitable representations for lines in this geometry.
</p><p>The "shortness" and "straightness" of a line, interpreted as the property that the <a href="/wiki/Distance" title="Distance">distance</a> along the line between any two of its points is minimized (see <a href="/wiki/Triangle_inequality" title="Triangle inequality">triangle inequality</a>), can be generalized and leads to the concept of <a href="/wiki/Geodesic" title="Geodesic">geodesics</a> in <a href="/wiki/Metric_space" title="Metric space">metric spaces</a>.
</p>

For other uses in mathematics, see <a class="mw-redirect mw-disambig" href="/wiki/Ray_(disambiguation)#Science_and_mathematics" title="Ray (disambiguation)">Ray (disambiguation) § Science and mathematics</a>.</div><div class="thumb tright"><div class="thumbinner" style="width:302px;"><a class="image" href="/wiki/File:Ray_(A,_B,_C).svg"><img alt="Ray" class="thumbimage" data-file-height="100" data-file-width="1000" decoding="async" height="30" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/8e/Ray_%28A%2C_B%2C_C%29.svg/300px-Ray_%28A%2C_B%2C_C%29.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/8e/Ray_%28A%2C_B%2C_C%29.svg/450px-Ray_%28A%2C_B%2C_C%29.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/8e/Ray_%28A%2C_B%2C_C%29.svg/600px-Ray_%28A%2C_B%2C_C%29.svg.png 2x" width="300"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Ray_(A,_B,_C).svg" title="Enlarge"></a></div>A ray with a terminus at A, with two points B and C on the right</div></div></div>
<p>Given a line and any point <i>A</i> on it, we may consider <i>A</i> as decomposing this line into two parts.
Each such part is called a <b>ray</b> and the point <i>A</i> is called its <i>initial point</i>. It is also known as half-line, a one-dimensional <a href="/wiki/Half-space_(geometry)" title="Half-space (geometry)">half-space</a>. The point A is considered to be a member of the ray.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> Intuitively, a ray consists of those points on a line passing through <i>A</i> and proceeding indefinitely, starting at <i>A</i>, in one direction only along the line. However, in order to use this concept of a ray in proofs a more precise definition is required.
</p><p>Given distinct points <i>A</i> and <i>B</i>, they determine a unique ray with initial point <i>A</i>. As two points define a unique line, this ray consists of all the points between <i>A</i> and <i>B</i> (including <i>A</i> and <i>B</i>) and all the points <i>C</i> on the line through <i>A</i> and <i>B</i> such that <i>B</i> is between <i>A</i> and <i>C</i>.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> This is, at times, also expressed as the set of all points <i>C</i> on the line determined by <i>A</i> and <i>B</i> such that <i>A</i> is not between <i>B</i> and <i>C</i>.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> A point <i>D</i>, on the line determined by <i>A</i> and <i>B</i> but not in the ray with initial point <i>A</i> determined by <i>B</i>, will determine another ray with initial point <i>A</i>. With respect to the <i>AB</i> ray, the <i>AD</i> ray is called the <i>opposite ray</i>.
</p><p>Thus, we would say that two different points, <i>A</i> and <i>B</i>, define a line and a decomposition of this line into the <a href="/wiki/Disjoint_union" title="Disjoint union">disjoint union</a> of an open segment <span class="texhtml">(<i>A</i>, <i>B</i>)</span> and two rays, <i>BC</i> and <i>AD</i> (the point <i>D</i> is not drawn in the diagram, but is to the left of <i>A</i> on the line <i>AB</i>). These are not opposite rays since they have different initial points.
</p><p>In Euclidean geometry two rays with a common endpoint form an <a href="/wiki/Angle" title="Angle">angle</a>.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup>
</p><p>The definition of a ray depends upon the notion of betweenness for points on a line. It follows that rays exist only for geometries for which this notion exists, typically <a href="/wiki/Euclidean_geometry" title="Euclidean geometry">Euclidean geometry</a> or <a href="/wiki/Affine_geometry" title="Affine geometry">affine geometry</a> over an <a href="/wiki/Ordered_field" title="Ordered field">ordered field</a>. On the other hand, rays do not exist in <a href="/wiki/Projective_geometry" title="Projective geometry">projective geometry</a> nor in a geometry over a non-ordered field, like the <a href="/wiki/Complex_number" title="Complex number">complex numbers</a> or any <a href="/wiki/Finite_field" title="Finite field">finite field</a>.
</p>
<h3><span class="mw-headline" id="Line_segment">Line segment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=16" title="Edit section: Line segment">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Line_segment" title="Line segment">Line segment</a></div>
<div class="thumb tright"><div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Adcinak.svg"><img alt="see caption" class="thumbimage" data-file-height="29" data-file-width="160" decoding="async" height="40" src="//upload.wikimedia.org/wikipedia/commons/thumb/6/65/Adcinak.svg/220px-Adcinak.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/6/65/Adcinak.svg/330px-Adcinak.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/6/65/Adcinak.svg/440px-Adcinak.svg.png 2x" width="220"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Adcinak.svg" title="Enlarge"></a></div>Drawing of a line segment "AB" on the line "a"</div></div></div>
<p>A <a href="/wiki/Line_segment" title="Line segment">line segment</a> is a part of a line that is bounded by two distinct end points and contains every point on the line between its end points. Depending on how the line segment is defined, either of the two end points may or may not be part of the line segment. Two or more line segments may have some of the same relationships as lines, such as being parallel, intersecting, or skew, but unlike lines they may be none of these, if they are <a class="mw-redirect" href="/wiki/Coplanar" title="Coplanar">coplanar</a> and either do not intersect or are <a href="/wiki/Collinearity" title="Collinearity">collinear</a>.
</p>
<h3><span class="mw-headline" id="Number_line">Number line</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Line_(geometry)&amp;action=edit&amp;section=17" title="Edit section: Number line">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Number_line" title="Number line">Number line</a></div>
<div class="thumb tright"><div class="thumbinner" style="width:302px;"><a class="image" href="/wiki/File:Number_line_with_x_smaller_than_y.svg"><img alt="see caption" class="thumbimage" data-file-height="79" data-file-width="457" decoding="async" height="52" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/77/Number_line_with_x_smaller_than_y.svg/300px-Number_line_with_x_smaller_than_y.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/7/77/Number_line_with_x_smaller_than_y.svg/450px-Number_line_with_x_smaller_than_y.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/7/77/Number_line_with_x_smaller_than_y.svg/600px-Number_line_with_x_smaller_than_y.svg.png 2x" width="300"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Number_line_with_x_smaller_than_y.svg" title="Enlarge"></a></div>A number line, with variable x on the left and y on the right. Therefore, x is smaller than y.</div></div></div>
<p>A point on number line corresponds to a <a href="/wiki/Real_number" title="Real number">real number</a> and vice versa.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> Usually, <a href="/wiki/Integer" title="Integer">integers</a> are evenly spaced on the line, with positive numbers are on the right, negative numbers on the left.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2022)">citation needed</span></a></i>]</sup> As an extension to the concept, an <a href="/wiki/Imaginary_line_(mathematics)" title="Imaginary line (mathematics)">imaginary line</a> representing <a href="/wiki/Imaginary_number" title="Imaginary number">imaginary numbers</a> can be drawn perpendicular to the number line at zero.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> The two lines forms the <a href="/wiki/Complex_plane" title="Complex plane">complex plane</a>, a geometrical representation of the set of <a href="/wiki/Complex_number" title="Complex number">complex numbers</a>.
</p>
