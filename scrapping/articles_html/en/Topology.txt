<h1 class="firstHeading mw-first-heading" id="firstHeading"><span class="mw-page-title-main">Topology</span></h1>

<div class="mw-body-content mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><div class="mw-parser-output"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Branch of mathematics</div>
<style data-mw-deduplicate="TemplateStyles:r1033289096">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}</style><div class="hatnote navigation-not-searchable" role="note">This article is about the branch of mathematics. For the mathematical structure, see <a href="/wiki/Topological_space" title="Topological space">Topological space</a>. For other uses, see <a class="mw-disambig" href="/wiki/Topology_(disambiguation)" title="Topology (disambiguation)">Topology (disambiguation)</a>.</div>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Not to be confused with <a href="/wiki/Topography" title="Topography">topography</a> or <a class="mw-redirect mw-disambig" href="/wiki/Typology_(disambiguation)" title="Typology (disambiguation)">typology</a>.</div>
<p class="mw-empty-elt">
</p>
<div class="thumb tright"><div class="thumbinner" style="width:242px;"><a class="image" href="/wiki/File:M%C3%B6bius_strip.jpg"><img alt="" class="thumbimage" data-file-height="824" data-file-width="1328" decoding="async" height="149" src="//upload.wikimedia.org/wikipedia/commons/thumb/d/d9/M%C3%B6bius_strip.jpg/240px-M%C3%B6bius_strip.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/d/d9/M%C3%B6bius_strip.jpg/360px-M%C3%B6bius_strip.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/d/d9/M%C3%B6bius_strip.jpg/480px-M%C3%B6bius_strip.jpg 2x" width="240"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:M%C3%B6bius_strip.jpg" title="Enlarge"></a></div><a href="/wiki/M%C3%B6bius_strip" title="Möbius strip">Möbius strips</a>, which have only one surface and one edge, are a kind of object studied in topology.</div></div></div>
<p>In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, <b>topology</b> (from the <a href="/wiki/Greek_language" title="Greek language">Greek</a> words <span title="Ancient Greek-language text"><span lang="grc"><a class="extiw" href="https://en.wiktionary.org/wiki/%CF%84%CF%8C%CF%80%CE%BF%CF%82" title="wikt:τόπος">τόπος</a></span></span>, 'place, location', and <span title="Greek-language text"><span lang="el">λόγος</span></span>, 'study') is concerned with the properties of a <a href="/wiki/Mathematical_object" title="Mathematical object">geometric object</a> that are preserved under <a href="/wiki/Continuous_function" title="Continuous function">continuous</a> <a class="mw-redirect" href="/wiki/Deformation_theory" title="Deformation theory">deformations</a>, such as <a href="/wiki/Stretch_factor" title="Stretch factor">stretching</a>, <a href="/wiki/Twist_(mathematics)" title="Twist (mathematics)">twisting</a>, crumpling, and bending; that is, without closing holes, opening holes, tearing, gluing, or passing through itself.
</p><p>A <a href="/wiki/Topological_space" title="Topological space">topological space</a> is a <a href="/wiki/Set_(mathematics)" title="Set (mathematics)">set</a> endowed with a structure, called a <i>topology</i>, which allows defining continuous deformation of subspaces, and, more generally, all kinds of <a class="mw-redirect" href="/wiki/Continuity_(mathematics)" title="Continuity (mathematics)">continuity</a>. <a href="/wiki/Euclidean_space" title="Euclidean space">Euclidean spaces</a>, and, more generally, <a href="/wiki/Metric_space" title="Metric space">metric spaces</a> are examples of a topological space, as any distance or metric defines a topology. The deformations that are considered in topology are <a href="/wiki/Homeomorphism" title="Homeomorphism">homeomorphisms</a> and <a href="/wiki/Homotopy" title="Homotopy">homotopies</a>. A property that is invariant under such deformations is a <a href="/wiki/Topological_property" title="Topological property">topological property</a>. Basic examples of topological properties are: the <a class="mw-redirect" href="/wiki/Topological_dimension" title="Topological dimension">dimension</a>, which allows distinguishing between a <a href="/wiki/Line_(geometry)" title="Line (geometry)">line</a> and a <a href="/wiki/Surface_(mathematics)" title="Surface (mathematics)">surface</a>; <a class="mw-redirect" href="/wiki/Compact_(topology)" title="Compact (topology)">compactness</a>, which allows distinguishing between a line and a circle; <a href="/wiki/Connectedness" title="Connectedness">connectedness</a>, which allows distinguishing a circle from two non-intersecting circles.
</p><p>The ideas underlying topology go back to <a class="mw-redirect" href="/wiki/Gottfried_Leibniz" title="Gottfried Leibniz">Gottfried Leibniz</a>, who in the 17th century envisioned the <span title="Latin-language text"><i lang="la">geometria situs</i></span> and <span title="Latin-language text"><i lang="la">analysis situs</i></span>. <a href="/wiki/Leonhard_Euler" title="Leonhard Euler">Leonhard Euler</a>'s <a href="/wiki/Seven_Bridges_of_K%C3%B6nigsberg" title="Seven Bridges of Königsberg">Seven Bridges of Königsberg</a> problem and <a href="/wiki/Euler_characteristic#Polyhedra" title="Euler characteristic">polyhedron formula</a> are arguably the field's first theorems. The term <i>topology</i> was introduced by <a href="/wiki/Johann_Benedict_Listing" title="Johann Benedict Listing">Johann Benedict Listing</a> in the 19th century, although it was not until the first decades of the 20th century that the idea of a topological space was developed.
</p>

<h2><span class="mw-headline" id="Motivation">Motivation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=1" title="Edit section: Motivation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The motivating insight behind topology is that some geometric problems depend not on the exact shape of the objects involved, but rather on the way they are put together. For example, the square and the circle have many properties in common: they are both one dimensional objects (from a topological point of view) and both separate the plane into two parts, the part inside and the part outside.
</p><p>In one of the first papers in topology, Leonhard Euler demonstrated that it was impossible to find a route through the town of Königsberg (now <a href="/wiki/Kaliningrad" title="Kaliningrad">Kaliningrad</a>) that would cross each of its seven bridges exactly once. This result did not depend on the lengths of the bridges or on their distance from one another, but only on connectivity properties: which bridges connect to which islands or riverbanks. This <a href="/wiki/Seven_Bridges_of_K%C3%B6nigsberg" title="Seven Bridges of Königsberg">Seven Bridges of Königsberg</a> problem led to the branch of mathematics known as <a href="/wiki/Graph_theory" title="Graph theory">graph theory</a>.
</p>

<p>Similarly, the <a href="/wiki/Hairy_ball_theorem" title="Hairy ball theorem">hairy ball theorem</a> of algebraic topology says that "one cannot comb the hair flat on a hairy ball without creating a <a href="/wiki/Cowlick" title="Cowlick">cowlick</a>." This fact is immediately convincing to most people, even though they might not recognize the more formal statement of the theorem, that there is no nonvanishing continuous <a href="/wiki/Vector_field" title="Vector field">tangent vector field</a> on the sphere. As with the <i>Bridges of Königsberg</i>, the result does not depend on the shape of the sphere; it applies to any kind of smooth blob, as long as it has no holes.
</p><p>To deal with these problems that do not rely on the exact shape of the objects, one must be clear about just what properties these problems <em>do</em> rely on. From this need arises the notion of homeomorphism. The impossibility of crossing each bridge just once applies to any arrangement of bridges homeomorphic to those in Königsberg, and the hairy ball theorem applies to any space homeomorphic to a sphere.
</p><p>Intuitively, two spaces are homeomorphic if one can be deformed into the other without cutting or gluing. A traditional joke is that a topologist cannot distinguish a coffee mug from a doughnut, since a sufficiently pliable doughnut could be reshaped to a coffee cup by creating a dimple and progressively enlarging it, while shrinking the hole into a handle.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>
</p><p>Homeomorphism can be considered the most basic <a href="/wiki/Homeomorphism" title="Homeomorphism">topological equivalence</a>. Another is <a class="mw-redirect" href="/wiki/Homotopy_equivalence" title="Homotopy equivalence">homotopy equivalence</a>. This is harder to describe without getting technical, but the essential notion is that two objects are homotopy equivalent if they both result from "squishing" some larger object.
</p>

<td style="vertical-align: top"><a class="image" href="/wiki/File:Alphabet_homeo.png"><img alt="{A,R} {B} {C,G,I,J,L,M,N,S,U,V,W,Z}, {D,O} {E,F,T,Y} {H,K}, {P,Q} {X}" data-file-height="87" data-file-width="378" decoding="async" height="62" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/74/Alphabet_homeo.png/270px-Alphabet_homeo.png" srcset="//upload.wikimedia.org/wikipedia/commons/7/74/Alphabet_homeo.png 1.5x" width="270"/></a>

<td style="vertical-align: top"><a class="image" href="/wiki/File:Alphabet_homotopy.png"><img alt="{A,R,D,O,P,Q} {B}, {C,E,F,G,H,I,J,K,L,M,N,S,T,U,V,W,X,Y,Z}" data-file-height="57" data-file-width="424" decoding="async" height="40" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/7c/Alphabet_homotopy.png/300px-Alphabet_homotopy.png" srcset="//upload.wikimedia.org/wikipedia/commons/7/7c/Alphabet_homotopy.png 1.5x" width="300"/></a>

<p>An introductory <a href="/wiki/Exercise_(mathematics)" title="Exercise (mathematics)">exercise</a> is to classify the uppercase letters of the <a href="/wiki/English_alphabet" title="English alphabet">English alphabet</a> according to homeomorphism and homotopy equivalence. The result depends on the font used, and on whether the strokes making up the letters have some thickness or are ideal curves with no thickness.  The figures here use the <a href="/wiki/Sans-serif" title="Sans-serif">sans-serif</a> <a class="mw-redirect" href="/wiki/Myriad_(font)" title="Myriad (font)">Myriad</a> font and are assumed to consist of ideal curves without thickness. Homotopy equivalence is a coarser relationship than homeomorphism; a homotopy equivalence class can contain several homeomorphism classes. The simple case of homotopy equivalence described above can be used here to show two letters are homotopy equivalent. For example, O fits inside P and the tail of the P can be squished to the "hole" part.
</p><p>Homeomorphism classes are:
</p>
<ul><li>no holes corresponding with C, G, I, J, L, M, N, S, U, V, W, and Z;</li>
<li>no holes and three tails corresponding with E, F, T, and Y;</li>
<li>no holes and four tails corresponding with X;</li>
<li>one hole and no tail corresponding with D and O;</li>
<li>one hole and one tail corresponding with P and Q;</li>
<li>one hole and two tails corresponding with A and R;</li>
<li>two holes and no tail corresponding with B; and</li>
<li>a bar with four tails corresponding with H and K; the "bar" on the <i>K</i> is almost too short to see.</li></ul>
<p>Homotopy classes are larger, because the tails can be squished down to a point. They are:
</p>
<ul><li>one hole,</li>
<li>two holes, and</li>
<li>no holes.</li></ul>
<p>To classify the letters correctly, we must show that two letters in the same class are equivalent and two letters in different classes are not equivalent. In the case of homeomorphism, this can be done by selecting points and showing their removal disconnects the letters differently. For example, X and Y are not homeomorphic because removing the center point of the X leaves four pieces; whatever point in Y corresponds to this point, its removal can leave at most three pieces. The case of homotopy equivalence is harder and requires a more elaborate argument showing an algebraic invariant, such as the <a href="/wiki/Fundamental_group" title="Fundamental group">fundamental group</a>, is different on the supposedly differing classes.
</p><p>Letter topology has practical relevance in <a href="/wiki/Stencil" title="Stencil">stencil</a> <a href="/wiki/Typography" title="Typography">typography</a>. For instance, <a href="/wiki/Braggadocio_(typeface)" title="Braggadocio (typeface)">Braggadocio</a> font stencils are made of one connected piece of material.
</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright"><div class="thumbinner" style="width:242px;"><a class="image" href="/wiki/File:Konigsberg_bridges.png"><img alt="" class="thumbimage" data-file-height="238" data-file-width="302" decoding="async" height="189" src="//upload.wikimedia.org/wikipedia/commons/thumb/5/5d/Konigsberg_bridges.png/240px-Konigsberg_bridges.png" srcset="//upload.wikimedia.org/wikipedia/commons/5/5d/Konigsberg_bridges.png 1.5x" width="240"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Konigsberg_bridges.png" title="Enlarge"></a></div>The <a href="/wiki/Seven_Bridges_of_K%C3%B6nigsberg" title="Seven Bridges of Königsberg">Seven Bridges of Königsberg</a> was a problem solved by Euler.</div></div></div>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">See also: <a href="/wiki/History_of_the_separation_axioms" title="History of the separation axioms">History of the separation axioms</a></div>
<p>Topology, as a well-defined mathematical discipline, originates in the early part of the twentieth century, but some isolated results can be traced back several centuries.<sup class="reference" id="cite_ref-Croom7_2-0"><a href="#cite_note-Croom7-2">[2]</a></sup> Among these are certain questions in geometry investigated by <a href="/wiki/Leonhard_Euler" title="Leonhard Euler">Leonhard Euler</a>. His 1736 paper on the <a href="/wiki/Seven_Bridges_of_K%C3%B6nigsberg" title="Seven Bridges of Königsberg">Seven Bridges of Königsberg</a> is regarded as one of the first practical applications of topology.<sup class="reference" id="cite_ref-Croom7_2-1"><a href="#cite_note-Croom7-2">[2]</a></sup> On 14 November 1750, Euler wrote to a friend that he had realized the importance of the <i>edges</i> of a <a href="/wiki/Polyhedron" title="Polyhedron">polyhedron</a>. This led to his <a class="mw-redirect" href="/wiki/Euler%27s_polyhedron_formula" title="Euler's polyhedron formula">polyhedron formula</a>, <span class="texhtml"><i>V</i> − <i>E</i> + <i>F</i> = 2</span> (where <span class="texhtml mvar" style="font-style:italic;">V</span>, <span class="texhtml mvar" style="font-style:italic;">E</span>, and <span class="texhtml mvar" style="font-style:italic;">F</span> respectively indicate the number of vertices, edges, and faces of the polyhedron). Some authorities regard this analysis as the first theorem, signaling the birth of topology.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>
</p><p>Further contributions were made by <a href="/wiki/Augustin-Louis_Cauchy" title="Augustin-Louis Cauchy">Augustin-Louis Cauchy</a>, <a href="/wiki/Ludwig_Schl%C3%A4fli" title="Ludwig Schläfli">Ludwig Schläfli</a>, <a href="/wiki/Johann_Benedict_Listing" title="Johann Benedict Listing">Johann Benedict Listing</a>, <a href="/wiki/Bernhard_Riemann" title="Bernhard Riemann">Bernhard Riemann</a> and <a href="/wiki/Enrico_Betti" title="Enrico Betti">Enrico Betti</a>.<sup class="reference" id="cite_ref-Richeson_4-0"><a href="#cite_note-Richeson-4">[4]</a></sup> Listing introduced the term "Topologie" in <i>Vorstudien zur Topologie</i>, written in his native German, in 1847, having used the word for ten years in correspondence before its first appearance in print.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> The English form "topology" was used in 1883 in Listing's obituary in the journal <a href="/wiki/Nature_(journal)" title="Nature (journal)"><i>Nature</i></a> to distinguish "qualitative geometry from the ordinary geometry in which quantitative relations chiefly are treated".<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>
</p><p>Their work was corrected, consolidated and greatly extended by <a href="/wiki/Henri_Poincar%C3%A9" title="Henri Poincaré">Henri Poincaré</a>. In 1895, he published his ground-breaking paper on <i><a href="/wiki/Analysis_Situs_(paper)" title="Analysis Situs (paper)">Analysis Situs</a></i>, which introduced the concepts now known as <a href="/wiki/Homotopy" title="Homotopy">homotopy</a> and <a href="/wiki/Homology_(mathematics)" title="Homology (mathematics)">homology</a>, which are now considered part of <a href="/wiki/Algebraic_topology" title="Algebraic topology">algebraic topology</a>.<sup class="reference" id="cite_ref-Richeson_4-1"><a href="#cite_note-Richeson-4">[4]</a></sup>
</p>

<p>Unifying the work on function spaces of <a href="/wiki/Georg_Cantor" title="Georg Cantor">Georg Cantor</a>, <a href="/wiki/Vito_Volterra" title="Vito Volterra">Vito Volterra</a>, <a href="/wiki/Cesare_Arzel%C3%A0" title="Cesare Arzelà">Cesare Arzelà</a>, <a href="/wiki/Jacques_Hadamard" title="Jacques Hadamard">Jacques Hadamard</a>, <a href="/wiki/Giulio_Ascoli" title="Giulio Ascoli">Giulio Ascoli</a> and others, <a class="mw-redirect" href="/wiki/Maurice_Fr%C3%A9chet" title="Maurice Fréchet">Maurice Fréchet</a> introduced the <a href="/wiki/Metric_space" title="Metric space">metric space</a> in 1906.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> A metric space is now considered a special case of a general topological space, with any given topological space potentially giving rise to many distinct metric spaces. In 1914, <a href="/wiki/Felix_Hausdorff" title="Felix Hausdorff">Felix Hausdorff</a> coined the term "topological space" and gave the definition for what is now called a <a href="/wiki/Hausdorff_space" title="Hausdorff space">Hausdorff space</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> Currently, a topological space is a slight generalization of Hausdorff spaces, given in 1922 by <a href="/wiki/Kazimierz_Kuratowski" title="Kazimierz Kuratowski">Kazimierz Kuratowski</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup>
</p><p>Modern topology depends strongly on the ideas of set theory, developed by Georg Cantor in the later part of the 19th century. In addition to establishing the basic ideas of set theory, Cantor considered point sets in <a href="/wiki/Euclidean_space" title="Euclidean space">Euclidean space</a> as part of his study of <a href="/wiki/Fourier_series" title="Fourier series">Fourier series</a>. For further developments, see <a class="mw-redirect" href="/wiki/Point-set_topology" title="Point-set topology">point-set topology</a> and algebraic topology.
</p><p>The 2022 <a href="/wiki/Abel_Prize" title="Abel Prize">Abel Prize</a> was awarded to <a href="/wiki/Dennis_Sullivan" title="Dennis Sullivan">Dennis Sullivan</a> "for his groundbreaking contributions to topology in its broadest sense, and in particular its algebraic, geometric and dynamical aspects".<sup class="reference" id="cite_ref-Abel-2022_10-0"><a href="#cite_note-Abel-2022-10">[10]</a></sup>
</p>
<h2><span class="mw-headline" id="Concepts">Concepts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=3" title="Edit section: Concepts">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Topologies_on_sets">Topologies on sets</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=4" title="Edit section: Topologies on sets">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Topological_space" title="Topological space">Topological space</a></div>
<p>The term <i>topology</i> also refers to a specific mathematical idea central to the area of mathematics called topology. Informally, a topology tells how elements of a set relate spatially to each other. The same set can have different topologies. For instance, the <a class="mw-redirect" href="/wiki/Real_line" title="Real line">real line</a>, the <a href="/wiki/Complex_plane" title="Complex plane">complex plane</a>, and the <a href="/wiki/Cantor_set" title="Cantor set">Cantor set</a> can be thought of as the same set with different topologies.
</p><p>Formally, let <span class="texhtml mvar" style="font-style:italic;">X</span> be a set and let <span class="texhtml mvar" style="font-style:italic;">τ</span> be a <a href="/wiki/Family_of_sets" title="Family of sets">family</a> of subsets of <span class="texhtml mvar" style="font-style:italic;">X</span>. Then <span class="texhtml mvar" style="font-style:italic;">τ</span> is called a topology on <span class="texhtml mvar" style="font-style:italic;">X</span> if:
</p>
<ol><li>Both the empty set and <span class="texhtml mvar" style="font-style:italic;">X</span> are elements of <span class="texhtml mvar" style="font-style:italic;">τ</span>.</li>
<li>Any union of elements of <span class="texhtml mvar" style="font-style:italic;">τ</span> is an element of <span class="texhtml mvar" style="font-style:italic;">τ</span>.</li>
<li>Any intersection of finitely many elements of <span class="texhtml mvar" style="font-style:italic;">τ</span> is an element of <span class="texhtml mvar" style="font-style:italic;">τ</span>.</li></ol>
<p>If <span class="texhtml mvar" style="font-style:italic;">τ</span> is a topology on <span class="texhtml mvar" style="font-style:italic;">X</span>, then the pair <span class="texhtml">(<i>X</i>, <i>τ</i>)</span> is called a topological space. The notation <span class="texhtml"><i>X</i><sub><i>τ</i></sub></span> may be used to denote a set <span class="texhtml mvar" style="font-style:italic;">X</span> endowed with the particular topology <span class="texhtml mvar" style="font-style:italic;">τ</span>. By definition, every topology is a <a href="/wiki/Pi-system" title="Pi-system"><span class="texhtml mvar" style="font-style:italic;">π</span>-system</a>.
</p><p>The members of <span class="texhtml mvar" style="font-style:italic;">τ</span> are called <i>open sets</i> in <span class="texhtml mvar" style="font-style:italic;">X</span>. A subset of <span class="texhtml mvar" style="font-style:italic;">X</span> is said to be closed if its complement is in <span class="texhtml mvar" style="font-style:italic;">τ</span> (that is, its complement is open). A subset of <span class="texhtml mvar" style="font-style:italic;">X</span> may be open, closed, both (a <a href="/wiki/Clopen_set" title="Clopen set">clopen set</a>), or neither. The empty set and <span class="texhtml mvar" style="font-style:italic;">X</span> itself are always both closed and open. An open subset of <span class="texhtml mvar" style="font-style:italic;">X</span> which contains a point <span class="texhtml mvar" style="font-style:italic;">x</span> is called a <a class="mw-redirect" href="/wiki/Neighborhood_(mathematics)" title="Neighborhood (mathematics)">neighborhood</a> of <span class="texhtml mvar" style="font-style:italic;">x</span>.
</p>
<h3><span class="mw-headline" id="Continuous_functions_and_homeomorphisms">Continuous functions and homeomorphisms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=5" title="Edit section: Continuous functions and homeomorphisms">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><span class="anchor" id="mug-and-doughnut"></span>
</p>
<div class="thumb tright"><div class="thumbinner" style="width:242px;"><a class="image" href="/wiki/File:Topology_joke.jpg"><img alt="" class="thumbimage" data-file-height="2886" data-file-width="3648" decoding="async" height="190" src="//upload.wikimedia.org/wikipedia/commons/thumb/d/d0/Topology_joke.jpg/240px-Topology_joke.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/d/d0/Topology_joke.jpg/360px-Topology_joke.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/d/d0/Topology_joke.jpg/480px-Topology_joke.jpg 2x" width="240"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Topology_joke.jpg" title="Enlarge"></a></div>A continuous transformation can turn a coffee mug into a donut.<br/>Ceramic model by Keenan Crane and <a href="/wiki/Henry_Segerman" title="Henry Segerman">Henry Segerman</a>.</div></div></div>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main articles: <a href="/wiki/Continuous_function" title="Continuous function">Continuous function</a> and <a href="/wiki/Homeomorphism" title="Homeomorphism">homeomorphism</a></div>
<p>A <a href="/wiki/Function_(mathematics)" title="Function (mathematics)">function</a> or map from one topological space to another is called <i>continuous</i> if the inverse image of any open set is open. If the function maps the <a href="/wiki/Real_number" title="Real number">real numbers</a> to the real numbers (both spaces with the standard topology), then this definition of continuous is equivalent to the definition of continuous in <a href="/wiki/Calculus" title="Calculus">calculus</a>. If a continuous function is <a href="/wiki/Injective_function" title="Injective function">one-to-one</a> and <a href="/wiki/Surjective_function" title="Surjective function">onto</a>, and if the inverse of the function is also continuous, then the function is called a homeomorphism and the domain of the function is said to be homeomorphic to the range. Another way of saying this is that the function has a natural extension to the topology. If two spaces are homeomorphic, they have identical topological properties, and are considered topologically the same. The cube and the sphere are homeomorphic, as are the coffee cup and the doughnut. But the circle is not homeomorphic to the doughnut.
</p>
<h3><span class="mw-headline" id="Manifolds">Manifolds</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=6" title="Edit section: Manifolds">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Manifold" title="Manifold">Manifold</a></div>
<p>While topological spaces can be extremely varied and exotic, many areas of topology focus on the more familiar class of spaces known as manifolds. A <i>manifold</i> is a topological space that resembles Euclidean space near each point.  More precisely, each point of an <span class="texhtml mvar" style="font-style:italic;">n</span>-dimensional manifold has a <a class="mw-redirect" href="/wiki/Neighborhood_(mathematics)" title="Neighborhood (mathematics)">neighborhood</a> that is <a class="mw-redirect" href="/wiki/Homeomorphic" title="Homeomorphic">homeomorphic</a> to the Euclidean space of dimension <span class="texhtml mvar" style="font-style:italic;">n</span>.  <a href="/wiki/Line_(geometry)" title="Line (geometry)">Lines</a> and <a href="/wiki/Circle" title="Circle">circles</a>, but not <a href="/wiki/Lemniscate" title="Lemniscate">figure eights</a>, are one-dimensional manifolds.  Two-dimensional manifolds are also called <a href="/wiki/Surface_(topology)" title="Surface (topology)">surfaces</a>, although not all <a href="/wiki/Surface_(mathematics)" title="Surface (mathematics)">surfaces</a> are manifolds.  Examples include the <a href="/wiki/Plane_(geometry)" title="Plane (geometry)">plane</a>, the sphere, and the torus, which can all be realized without self-intersection in three dimensions, and the <a href="/wiki/Klein_bottle" title="Klein bottle">Klein bottle</a> and <a href="/wiki/Real_projective_plane" title="Real projective plane">real projective plane</a>, which cannot (that is, all their realizations are surfaces that are not manifolds).
</p>
<h2><span class="mw-headline" id="Topics">Topics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=7" title="Edit section: Topics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="General_topology">General topology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=8" title="Edit section: General topology">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/General_topology" title="General topology">General topology</a></div>
<p>General topology is the branch of topology dealing with the basic set-theoretic definitions and constructions used in topology.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup><sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> It is the foundation of most other branches of topology, including differential topology, geometric topology, and algebraic topology. Another name for general topology is point-set topology.
</p><p>The basic object of study is <a href="/wiki/Topological_space" title="Topological space">topological spaces</a>, which are sets equipped with a <a class="mw-redirect" href="/wiki/Topology_(structure)" title="Topology (structure)">topology</a>, that is, a family of <a href="/wiki/Subset" title="Subset">subsets</a>, called <i>open sets</i>, which is <a href="/wiki/Closure_(mathematics)" title="Closure (mathematics)">closed</a> under finite <a class="mw-redirect" href="/wiki/Set_intersection" title="Set intersection">intersections</a> and (finite or infinite) <a class="mw-redirect" href="/wiki/Set_union" title="Set union">unions</a>. The fundamental concepts of topology, such as <i><a class="mw-redirect" href="/wiki/Continuity_(mathematics)" title="Continuity (mathematics)">continuity</a></i>, <i><a class="mw-redirect" href="/wiki/Compactness" title="Compactness">compactness</a></i>, and <i><a href="/wiki/Connectedness" title="Connectedness">connectedness</a></i>, can be defined in terms of open sets. Intuitively, continuous functions take nearby points to nearby points. Compact sets are those that can be covered by finitely many sets of arbitrarily small size. Connected sets are sets that cannot be divided into two pieces that are far apart. The words <i>nearby</i>, <i>arbitrarily small</i>, and <i>far apart</i> can all be made precise by using open sets. Several topologies can be defined on a given space. Changing a topology consists of changing the collection of open sets. This changes which functions are continuous and which subsets are compact or connected.
</p><p><a href="/wiki/Metric_space" title="Metric space">Metric spaces</a> are an important class of topological spaces where the distance between any two points is defined by a function called a <i>metric</i>. In a metric space, an open set is a union of open disks, where an open disk of radius <span class="texhtml mvar" style="font-style:italic;">r</span> centered at <span class="texhtml mvar" style="font-style:italic;">x</span> is the set of all points whose distance to <span class="texhtml mvar" style="font-style:italic;">x</span> is less than <span class="texhtml mvar" style="font-style:italic;">r</span>. Many common spaces are topological spaces whose topology can be defined by a metric. This is the case of the <a class="mw-redirect" href="/wiki/Real_line" title="Real line">real line</a>, the <a href="/wiki/Complex_plane" title="Complex plane">complex plane</a>, real and complex <a href="/wiki/Vector_space" title="Vector space">vector spaces</a> and <a href="/wiki/Euclidean_space" title="Euclidean space">Euclidean spaces</a>. Having a metric simplifies many proofs.
</p>
<h3><span class="mw-headline" id="Algebraic_topology">Algebraic topology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=9" title="Edit section: Algebraic topology">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Algebraic_topology" title="Algebraic topology">Algebraic topology</a></div>
<p>Algebraic topology is a branch of mathematics that uses tools from <a href="/wiki/Algebra" title="Algebra">algebra</a> to study topological spaces.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> The basic goal is to find algebraic invariants that <a href="/wiki/Classification_theorem" title="Classification theorem">classify</a> topological spaces <a href="/wiki/Up_to" title="Up to">up to</a> homeomorphism, though usually most classify up to homotopy equivalence.
</p><p>The most important of these invariants are <a href="/wiki/Homotopy_group" title="Homotopy group">homotopy groups</a>, homology, and <a href="/wiki/Cohomology" title="Cohomology">cohomology</a>.
</p><p>Although algebraic topology primarily uses algebra to study topological problems, using topology to solve algebraic problems is sometimes also possible. Algebraic topology, for example, allows for a convenient proof that any subgroup of a <a href="/wiki/Free_group" title="Free group">free group</a> is again a free group.
</p>
<h3><span class="mw-headline" id="Differential_topology">Differential topology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=10" title="Edit section: Differential topology">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Differential_topology" title="Differential topology">Differential topology</a></div>
<p>Differential topology is the field dealing with <a href="/wiki/Differentiable_function" title="Differentiable function">differentiable functions</a> on <a href="/wiki/Differentiable_manifold" title="Differentiable manifold">differentiable manifolds</a>.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> It is closely related to <a href="/wiki/Differential_geometry" title="Differential geometry">differential geometry</a> and together they make up the geometric theory of differentiable manifolds.
</p><p>More specifically, differential topology considers the properties and structures that require only a <a href="/wiki/Smooth_structure" title="Smooth structure">smooth structure</a> on a manifold to be defined. Smooth manifolds are "softer" than manifolds with extra geometric structures, which can act as obstructions to certain types of equivalences and <a class="mw-redirect" href="/wiki/Deformation_theory" title="Deformation theory">deformations</a> that exist in differential topology. For instance, volume and <a class="mw-redirect" href="/wiki/Riemannian_curvature" title="Riemannian curvature">Riemannian curvature</a> are invariants that can distinguish different geometric structures on the same smooth manifold—that is, one can smoothly "flatten out" certain manifolds, but it might require distorting the space and affecting the curvature or volume.
</p>
<h3><span class="mw-headline" id="Geometric_topology">Geometric topology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=11" title="Edit section: Geometric topology">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<link href="mw-data:TemplateStyles:r1033289096" rel="mw-deduplicated-inline-style"/><div class="hatnote navigation-not-searchable" role="note">Main article: <a href="/wiki/Geometric_topology" title="Geometric topology">Geometric topology</a></div>
<p>Geometric topology is a branch of topology that primarily focuses on low-dimensional <a href="/wiki/Manifold" title="Manifold">manifolds</a> (that is, spaces of dimensions 2, 3, and 4) and their interaction with geometry, but it also includes some higher-dimensional topology.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> Some examples of topics in geometric topology are <a class="mw-redirect" href="/wiki/Orientable_manifold" title="Orientable manifold">orientability</a>, <a href="/wiki/Handle_decomposition" title="Handle decomposition">handle decompositions</a>, <a href="/wiki/Local_flatness" title="Local flatness">local flatness</a>, crumpling and the planar and higher-dimensional <a class="mw-redirect" href="/wiki/Jordan-Sch%C3%B6nflies_theorem" title="Jordan-Schönflies theorem">Schönflies theorem</a>.
</p><p>In high-dimensional topology, <a class="mw-redirect" href="/wiki/Characteristic_classes" title="Characteristic classes">characteristic classes</a> are a basic invariant, and <a href="/wiki/Surgery_theory" title="Surgery theory">surgery theory</a> is a key theory.
</p><p>Low-dimensional topology is strongly geometric, as reflected in the <a href="/wiki/Uniformization_theorem" title="Uniformization theorem">uniformization theorem</a> in 2 dimensions – every surface admits a constant curvature metric; geometrically, it has one of 3 possible geometries: positive <a href="/wiki/Curvature" title="Curvature">curvature</a>/spherical, zero curvature/flat, and negative curvature/hyperbolic – and the <a href="/wiki/Geometrization_conjecture" title="Geometrization conjecture">geometrization conjecture</a> (now theorem) in 3 dimensions – every 3-manifold can be cut into pieces, each of which has one of eight possible geometries.
</p><p>2-dimensional topology can be studied as <a href="/wiki/Complex_geometry" title="Complex geometry">complex geometry</a> in one variable (<a href="/wiki/Bernhard_Riemann" title="Bernhard Riemann">Riemann</a> surfaces are complex curves) – by the uniformization theorem every <a href="/wiki/Conformal_geometry" title="Conformal geometry">conformal class</a> of <a class="mw-redirect" href="/wiki/Metric_(mathematics)" title="Metric (mathematics)">metrics</a> is equivalent to a unique complex one, and 4-dimensional topology can be studied from the point of view of complex geometry in two variables (complex surfaces), though not every 4-manifold admits a complex structure.
</p>
<h3><span class="mw-headline" id="Generalizations">Generalizations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=12" title="Edit section: Generalizations">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Occasionally, one needs to use the tools of topology but a "set of points" is not available. In <a href="/wiki/Pointless_topology" title="Pointless topology">pointless topology</a> one considers instead the <a href="/wiki/Lattice_(order)" title="Lattice (order)">lattice</a> of open sets as the basic notion of the theory,<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> while <a href="/wiki/Grothendieck_topology" title="Grothendieck topology">Grothendieck topologies</a> are structures defined on arbitrary <a href="/wiki/Category_theory" title="Category theory">categories</a> that allow the definition of <a href="/wiki/Sheaf_(mathematics)" title="Sheaf (mathematics)">sheaves</a> on those categories, and with that the definition of general cohomology theories.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup>
</p>
<h2><span class="mw-headline" id="Applications">Applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=13" title="Edit section: Applications">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Biology">Biology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=14" title="Edit section: Biology">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Topology has been used to study various biological systems including molecules and nanostructure (e.g., membraneous objects<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup>). In particular, <a href="/wiki/Circuit_topology" title="Circuit topology">circuit topology</a> and <a href="/wiki/Knot_theory" title="Knot theory">knot theory</a> have been extensively applied to classify and compare the topology of folded proteins and nucleic acids. <a href="/wiki/Circuit_topology" title="Circuit topology">Circuit topology</a> classifies folded molecular chains based on the pairwise arrangement of their intra-chain contacts and chain crossings. <a href="/wiki/Knot_theory" title="Knot theory">Knot theory</a>, a branch of topology, is used in biology to study the effects of certain enzymes on DNA. These enzymes cut, twist, and reconnect the DNA, causing knotting with observable effects such as slower <a href="/wiki/Electrophoresis" title="Electrophoresis">electrophoresis</a>.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> Topology is also used in <a href="/wiki/Evolutionary_biology" title="Evolutionary biology">evolutionary biology</a> to represent the relationship between <a href="/wiki/Phenotype" title="Phenotype">phenotype</a> and <a href="/wiki/Genotype" title="Genotype">genotype</a>.<sup class="reference" id="cite_ref-bstadler2001_20-0"><a href="#cite_note-bstadler2001-20">[20]</a></sup> Phenotypic forms that appear quite different can be separated by only a few mutations depending on how genetic changes map to phenotypic changes during development. In neuroscience, topological quantities like the Euler characteristic and Betti number have been used to measure the complexity of patterns of activity in neural networks.
</p>
<h3><span class="mw-headline" id="Computer_science">Computer science</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=15" title="Edit section: Computer science">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Topological_data_analysis" title="Topological data analysis">Topological data analysis</a> uses techniques from algebraic topology to determine the large scale structure of a set (for instance, determining if a cloud of points is spherical or <a href="/wiki/Torus" title="Torus">toroidal</a>). The main method used by topological data analysis is to:
</p>
<ol><li>Replace a set of data points with a family of <a href="/wiki/Simplicial_complex" title="Simplicial complex">simplicial complexes</a>, indexed by a proximity parameter.</li>
<li>Analyse these topological complexes via algebraic topology – specifically, via the theory of <a href="/wiki/Persistent_homology" title="Persistent homology">persistent homology</a>.<sup class="reference" id="cite_ref-carlsson2009_21-0"><a href="#cite_note-carlsson2009-21">[21]</a></sup></li>
<li>Encode the persistent homology of a data set in the form of a parameterized version of a <a href="/wiki/Betti_number" title="Betti number">Betti number</a>, which is called a barcode.<sup class="reference" id="cite_ref-carlsson2009_21-1"><a href="#cite_note-carlsson2009-21">[21]</a></sup></li></ol>
<p>Several branches of <a class="mw-redirect" href="/wiki/Programming_language_semantics" title="Programming language semantics">programming language semantics</a>, such as <a href="/wiki/Domain_theory" title="Domain theory">domain theory</a>, are formalized using topology. In this context, <a href="/wiki/Steve_Vickers_(computer_scientist)" title="Steve Vickers (computer scientist)">Steve Vickers</a>, building on work by <a href="/wiki/Samson_Abramsky" title="Samson Abramsky">Samson Abramsky</a> and <a class="new" href="/w/index.php?title=Michael_B._Smyth&amp;action=edit&amp;redlink=1" title="Michael B. Smyth (page does not exist)">Michael B. Smyth</a>, characterizes topological spaces as <a href="/wiki/Boolean_algebra_(structure)" title="Boolean algebra (structure)">Boolean</a> or <a href="/wiki/Heyting_algebra" title="Heyting algebra">Heyting algebras</a> over open sets, which are characterized as <a class="mw-redirect" href="/wiki/Semidecidable" title="Semidecidable">semidecidable</a> (equivalently, finitely observable) properties.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup>
</p>
<h3><span class="mw-headline" id="Physics">Physics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=16" title="Edit section: Physics">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Topology is relevant to physics in areas such as <a href="/wiki/Condensed_matter_physics" title="Condensed matter physics">condensed matter physics</a>,<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> <a href="/wiki/Quantum_field_theory" title="Quantum field theory">quantum field theory</a> and <a href="/wiki/Physical_cosmology" title="Physical cosmology">physical cosmology</a>.
</p><p>The topological dependence of mechanical properties in solids is of interest in disciplines of <a href="/wiki/Mechanical_engineering" title="Mechanical engineering">mechanical engineering</a> and <a href="/wiki/Materials_science" title="Materials science">materials science</a>. Electrical and mechanical properties depend on the arrangement and network structures of <a class="mw-redirect" href="/wiki/Molecules" title="Molecules">molecules</a> and elementary units in materials.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> The <a href="/wiki/Compressive_strength" title="Compressive strength">compressive strength</a> of <a href="/wiki/Crumpling" title="Crumpling">crumpled</a> topologies is studied in attempts to understand the high strength to weight of such structures that are mostly empty space.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> Topology is of further significance in <a href="/wiki/Contact_mechanics" title="Contact mechanics">Contact mechanics</a> where the dependence of stiffness and friction on the <a href="/wiki/Fractal_dimension" title="Fractal dimension">dimensionality</a> of surface structures is the subject of interest with applications in multi-body physics.
</p><p>A <a href="/wiki/Topological_quantum_field_theory" title="Topological quantum field theory">topological quantum field theory</a> (or topological field theory or TQFT) is a quantum field theory that computes <a class="mw-redirect" href="/wiki/Topological_invariant" title="Topological invariant">topological invariants</a>.
</p><p>Although TQFTs were invented by physicists, they are also of mathematical interest, being related to, among other things, <a href="/wiki/Knot_theory" title="Knot theory">knot theory</a>, the theory of <a class="mw-redirect" href="/wiki/Four-manifold" title="Four-manifold">four-manifolds</a> in algebraic topology, and to the theory of <a class="mw-redirect" href="/wiki/Moduli_spaces" title="Moduli spaces">moduli spaces</a> in algebraic geometry. <a href="/wiki/Simon_Donaldson" title="Simon Donaldson">Donaldson</a>, <a href="/wiki/Vaughan_Jones" title="Vaughan Jones">Jones</a>, <a href="/wiki/Edward_Witten" title="Edward Witten">Witten</a>, and <a href="/wiki/Maxim_Kontsevich" title="Maxim Kontsevich">Kontsevich</a> have all won <a href="/wiki/Fields_Medal" title="Fields Medal">Fields Medals</a> for work related to topological field theory.
</p><p>The topological classification of <a href="/wiki/Calabi%E2%80%93Yau_manifold" title="Calabi–Yau manifold">Calabi–Yau manifolds</a> has important implications in <a href="/wiki/String_theory" title="String theory">string theory</a>, as different manifolds can sustain different kinds of strings.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup>
</p><p>In cosmology, topology can be used to describe the overall <a href="/wiki/Shape_of_the_universe" title="Shape of the universe">shape of the universe</a>.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> This area of research is commonly known as <a href="/wiki/Spacetime_topology" title="Spacetime topology">spacetime topology</a>.
</p><p>In condensed matter a relevant application to topological physics comes from the possibility to obtain one-way current, which is a current protected from backscattering. It was first discovered in electronics with the famous <a href="/wiki/Quantum_Hall_effect" title="Quantum Hall effect">quantum Hall effect</a>, and then generalized in other areas of physics, for instance in photonics<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup> by <a href="/wiki/Duncan_Haldane" title="Duncan Haldane">F.D.M Haldane</a>.
</p>
<h3><span class="mw-headline" id="Robotics">Robotics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=17" title="Edit section: Robotics">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The possible positions of a <a href="/wiki/Robot" title="Robot">robot</a> can be described by a <a href="/wiki/Manifold" title="Manifold">manifold</a> called <a href="/wiki/Configuration_space_(physics)" title="Configuration space (physics)">configuration space</a>.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> In the area of <a href="/wiki/Motion_planning" title="Motion planning">motion planning</a>, one finds paths between two points in configuration space. These paths represent a motion of the robot's <a href="/wiki/Joint" title="Joint">joints</a> and other parts into the desired pose.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup>
</p>
<h3><span class="mw-headline" id="Games_and_puzzles">Games and puzzles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=18" title="Edit section: Games and puzzles">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Disentanglement_puzzle" title="Disentanglement puzzle">Tanglement puzzles</a> are based on topological aspects of the puzzle's shapes and components.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup>
</p>
<h3><span class="mw-headline" id="Fiber_art">Fiber art</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Topology&amp;action=edit&amp;section=19" title="Edit section: Fiber art">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In order to create a continuous join of pieces in a modular construction, it is necessary to create an unbroken path in an order which surrounds each piece and traverses each edge only once. This process is an application of the <a href="/wiki/Eulerian_path" title="Eulerian path">Eulerian path</a>.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup>
</p>
